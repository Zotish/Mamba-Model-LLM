# MambaLM — Attention-Free Language Model

A small language model built **from scratch** using the Mamba SSM (Selective State Space Model) architecture, trained on the Stanford Alpaca instruction dataset, served via a FastAPI backend and React frontend.

---

## What is This?

This project builds a complete LLM pipeline from zero:

1. **Model** — Custom Mamba architecture (no Transformers, no Attention)
2. **Training** — Kaggle GPU (T4 x2), Alpaca 52K dataset, auto-save to HuggingFace
3. **Backend** — FastAPI server that loads the trained checkpoint
4. **Frontend** — React + Vite chat UI

---

## Model Architecture — MambaLM

### Core Idea
Traditional LLMs (GPT, LLaMA, Gemini) are built on the **Transformer** architecture, which uses **Self-Attention** — every token looks at every other token. This has O(L²) cost: double the sequence length, quadruple the compute.

MambaLM replaces attention entirely with a **Selective State Space Model (SSM)** — a learnable recurrence that runs in **O(L) linear time**.

### Architecture Per Layer

```
Input (B, L, d_model)
    │
  RMSNorm
    │
  MambaBlock:
    ├── in_proj        → splits into [x_ssm, z_gate]
    ├── DepthwiseConv1d(x_ssm)   ← local context mixing
    ├── Selective SSM scan(x_ssm) → y
    ├── y * SiLU(z_gate)          ← gating
    └── out_proj
    │
  + residual
    │
(repeat n_layer times)
    │
RMSNorm → LM Head (vocab projection)
```

### How the SSM Works

Unlike an RNN with fixed weights, the SSM's transition matrices **A, B, C** are **input-dependent** — they change for every token based on what the input is. This is called "selective" because the model learns to selectively remember or forget information based on content.

Key equations (discretized):
```
h[t] = dA[t] * h[t-1] + dB[t] * x[t]   ← state update
y[t] = C[t] · h[t] + D * x[t]           ← output
```

Where `dA`, `dB` are computed from the input at each step.

## Training

### Dataset
**Stanford Alpaca** — 52,000 instruction-following examples generated by GPT-3.5.

Format:
```
### Instruction:
What is the capital of France?

### Response:
The capital of France is Paris.
```

### Key Training Techniques

**1. Response-Only Loss**
Instead of computing loss on the full sequence (instruction + response), we compute loss **only on the response tokens**. This forces the model to focus on generating good answers, not predicting the instruction itself.

```python
if response_mask is not None:
    loss = F.cross_entropy(logits[mask], targets[mask])  # response tokens only
else:
    loss = F.cross_entropy(logits, targets)              # full sequence
```

**2. Cosine Learning Rate with Warmup**
```
Warmup: 0 → 3e-4  (first 300 steps)
Cosine decay: 3e-4 → 3e-5  (remaining steps)
```

**3. Gradient Accumulation**
Effective batch size = `BATCH_SIZE × ACCUM × num_GPUs` = `2 × 8 × 2 = 32`

**4. Auto-Save to HuggingFace Hub**
Every 300 steps, checkpoint is saved to `Zotish2002/mamba-alpaca` on HuggingFace. If the Kaggle session resets, training auto-resumes from the latest checkpoint.

**5. Multi-GPU (DataParallel)**
Runs on T4 x2 on Kaggle. `loss.mean()` called before backward to handle per-GPU loss tensors.

## How It Works (Full Pipeline)

```
User types question in browser
        ↓
React frontend (App.jsx)
  → formats as Alpaca prompt:
    "### Instruction:\n{question}\n\n### Response:\n"
        ↓
POST /generate → FastAPI backend (main.py)
        ↓
MambaRuntime (mamba_runtime.py)
  → loads ckpt.pt + meta.json
  → tokenizes with character-level vocab (421 chars)
  → runs MambaLM.forward() → logits
  → samples next token (temperature + top-k)
  → stops at "\n### Instruction:" tag
        ↓
Response sent back → displayed in chat UI
```

### What Makes This Unique

1. **Fully from scratch** — no HuggingFace `transformers`, no pretrained weights, no library model classes. Every layer written manually in PyTorch.

2. **Attention-free** — uses Selective SSM instead of self-attention. Theoretically faster for long sequences.

3. **Tiny but instruction-tuned** — 2.6M parameters trained specifically on instruction-following data (Alpaca format), not just raw text prediction.

4. **Character-level vocabulary** — vocab of 421 characters instead of 50K+ BPE tokens. Simple, no tokenizer library needed.

5. **Complete pipeline** — model → training → serving → UI, all in one project.
